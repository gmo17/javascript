Definition

JavavScript is programming language initially created to cater for interactive webpages including online games, it is primary used in conjunction with HTML and CSS for front-end development.

All modern web browsers do support JavavScript without requiring plug-ins. JavavScript purpose is to follow instructions defined by the programmer in a pre-defined scenario.

JavavScript is a:
- High-level (machine language), are languages that comprise variables, arrays, objects, complex arithmetic, boolean expressions, functions, loops, threads and locks.
- Dynamic, types are associated to values e.g. a variable definition can evolve. It can first be defined as a string then as a number.
- Weakly typed, are more lenient then strongly typed languages which usually generate an error or refuse to compile if the expected type doesn't match.
- Prototype-based, it is the action of inheriting from other objects but also maintaining its own attributes.
- Multi-paradigm, is simply a way to classify programming languages.

1|1a - Syntax


A programming syntax follows the same rule of any modern language, a syntax should should always follow grammatical and structural rules of a specific language. JavaScript syntax can be assimilated to programming languages such as Java or C++.

1|1b - Statement


A program or script is constituted by series of instructions known as statements. In order for JavaScript to be able to interpret these statements they must follow all syntax rules. A statement can be equated to a sentence in a modern language. Each statement should be placed on a separate line or separated by semicolon(s). It is generally accepted that each statement is placed on line and punctuated with a semicolon for better readability.

1|1d - Variables


A variables is a memory holder, a variable is not fixed unlike string they can be changed. First they have to be declared i.e. given a name with or without a value null. The act of giving it a value is known as assigning, once assigned the program can refer to a particular variable's value by simply calling it name. This will point to the value it has been allocated.

The keyword var can hold multiple variable declarations with a comma separator.
var name = "aName", age = 100; //ect...
Very useful in large programs to reduce lines count.

Variables naming convention:
- Variable names are case-sensitive, "name" or "Name" will be considered as two different variables.
var name = "aName", age = 100;
var Name = "aName", age = 100;

- Variables names should not contain space, the space can be replaced by an underscore.
var a name = "aName", age = 100; //not valid
var a_name = "aName", age = 100; //valid

- Variables can only start with "_" or "$"
var _name = "aName", age = 100; //valid
var $name = "aName", age = 100; //valid

Variables can be declared in multiple ways.

var x = null; //x has been declared with a null value.

x = 5; //x contains/assigned 5

var y = x * 2; //x is used in an expression


console.log(x);
output// 5

Variable Scope

Variables can be declared locally or globally i.e. defining their scope:
Locally - When declared within a block of code it can only be used (visible) within that.
Globally - If declared outside, it can be referenced anywhere in the script.

1|2 - PROGRAM STRUCTURE

1|2a - Control Flow

The control flow is present in many programming languages and JavavScript obeys the same rule.
It allow as the name implies to control the flow of execution a script. This is widely used in user input where conditions are set based on input.

Control Statements:

- if - If a condition is true execute the block of code within it. An if only execute once
var x = 5;
if(x > 4){
console.log("x is greater than 4");
}

output// x is greater than 4


- else - Else execute the alternative script, often if the if produce a false outcome.

var x = 5;
if(x < 4){
console.log("x is greater than 4");
}else{
console.log("x is not greater than 4");
}

output// x is not greater than 4

- else if - Is used to define a sequence of options.

var x = 5;
if(x < 4){
console.log("x is less than 4");
}else if(x == 5){
console.log("x is equal to 5");
}else if(x > 6){
console.log("x is greater than 5");
}

output// x is equal to 5

- switch - Similar to an if statement but condition takes more than on value.

switch(sunny){
case "sunny":
console.log("T-shirt day!");
break;

case "windy":
console.log("Coat day!");
break;

case "rainy":
console.log("Umbrella day!");
break;

default:
console.log("It is a Martian weather");
break;
}

output//
It is a T-shirt day

Loops
- while - Again, similar to an if however instead of executing only once it loop through as long the statement is true.
A while loop is when the number of iterations to get true is unknown in advance.

var x = 0;
while(x < 3){
console.log(x);
x++; }

output//
0
1
2

- for - The for loop is identical to the while, the only difference is that the for loop is more compact.
In a scenario where code minification is required the for statement is preferable as it reduces the number of lines required to do the same job.
In a loop the condition is check at each iteration and if not true the count is incremented.

for(var x = 0; x < 3; x++){
console.log(x);
}

output//
0
1
2

- do...while - differs from the while loop as the condition is tested at the end.
The do...while always execute the statement it encloses at least once.

do{
var aName = prompt("Enter your name!")
}while (!aName);
console.log(aName);

output// whatever entered

Unless the field is not blank it will keep asking for a name until provided.

- break - There are two ways to jump out of a statement, by producing a false output or by using the break statement.

for(var current = 20; ; current++){
if(current % 7 == 0)
break;
}
console.log(current);

output//
21

A function is a piece of code that groups related statements, with the perspective to define a single reusable task.
Wherever the same definition is required within the program the function can be invoked (called)

Calling an undefined function will result in an error message being frown out.

The naming convention stipulate that:
- Two functions can't have the same name
- It can contain letters, underscores and digit, but can't start with a digit
