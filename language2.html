Definition

JavavScript is programming language initially created to cater for interactive webpages including online games, it is primary used in conjunction with HTML and CSS for front-end development.

All modern web browsers do support JavavScript without requiring plug-ins. JavavScript purpose is to follow instructions defined by the programmer in a pre-defined scenario.

JavavScript is a:
- High-level (machine language), are languages that comprise variables, arrays, objects, complex arithmetic, boolean expressions, functions, loops, threads and locks.
- Dynamic, types are associated to values e.g. a variable definition can evolve. It can first be defined as a string then as a number.
- Weakly typed, are more lenient then strongly typed languages which usually generate an error or refuse to compile if the expected type doesn't match.
- Prototype-based, it is the action of inheriting from other objects but also maintaining its own attributes.
- Multi-paradigm, is simply a way to classify programming languages.

"Common programming paradigms include:
imperative which allows side effects, functional which disallows side effects, declarative which does not state the order in which operations execute, object-oriented which groups code together with the state the code modifies, procedural which groups code into functions, logic which has a particular style of execution model coupled to a particular style of syntax and grammar, and symbolic programming which has a particular style of syntax and grammar."

- Interpreted, is a programming language that doesn't require compilation in order to run.

Call Stack

Thread wise Javascript can only handle one task at a time and being a single threaded and concurrent language a solution has to be put in place to handle more tasks. A system that uses a combination of stacks, heaps and queues was put in place in order to run more than one stack at a time.


The call stack keeps track of all function calls specially for scripts calling several functions. Which is the current function being run and which one will be next. being runa call will be pushed into the stack and a return will be popped off from the top. Stack storing requires space in the computer's memory and if it take more space tham allocated a "out of stack" or "too much recursion" error will be thrown out. The order functions are being called in (first in last out) and what variable are they being called with.

Heaps


The heap is where objects and variable's memory is allocated.

Queue


A queue folows the logic of first in first ou, it maintains the order in which tasks are added to the queue and how they should be processed. Enqueue adds elements to the begining or end of the queue. Dequeue pops out elements from the begining of the queue Length is the length of the queue



Closure

Closure defines the ability to reference a specific instance of a local variable outside the function it is enclosed in. The variable instance can still be access even if the parent function has completed executing. These local variables are re-created on each function call and each call is unique, see example 2. Objects in a closure are given data privacy as functions defined outside the closure won't be able to access enclosed objects.

The child function as access to the parent scope (variables and parameters). The parent's function parameter acts a local variable, see example 3.

                          var book = 'Javascript';

                          return displayBook;

                        myCat();
                      

                      
                        //Example 2

                          var localVariable = n;
                          return function(){
                            return localVariable;
                          };

                        console.log(wrap1());
                        //ouput: 1

                        var wrap2 = wrapValue(2);
                        console.log(wrap2());
                        //ouput: 2
                      

                      
                        //Example 3

                          return function(n){
                            return n * factor;
                          };

                        //ouput: 10


Recursion

Recursion is a function that calls itself as long the stack is not overflow the stack. A recursion encompasses two cases a base case (returns a value) and recursive case (self-call).

In the example the factorial() function calls itself several times with different arguments to achieve the required result.
The only caviat is that self function calls will slow down the execution of the script compare to a simple loop (speed vs codd eleguance). 


                    
                      function func(){
                        if(/*base case*/){
                          return something;
                        }else{/*recursive case*/
                          func();
                        }
                      }

                      function factorial(n){
                        if(n == 1){
                          return n;
                        }else{
                          return n * factorial(n -1);
                        }
                      }
                      factorial(4);
