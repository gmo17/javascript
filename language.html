Definition

JavavScript is programming language initially created to cater for interactive webpages including online games, it is primary used in conjunction with HTML and CSS for front-end development.

All modern web browsers do support JavavScript without requiring plug-ins. JavavScript purpose is to follow instructions defined by the programmer in a pre-defined scenario.

JavavScript include the following:
- High-level (machine language), such as languages that have variables, arrays, objects, complex arithmetic, boolean expressions, functions, loops, threads and locks.
- Dynamic, when types are associated to values e.g. a variable definition can evolve, it can first be defined as a string and subsequently as a number.
- Weakly typed, will not generate an error or not compile if the expected type doesn't match.
- Prototype-based, the ability to inherit from other objects but also maintaining its own attributes.
- Interpreted, doesn't require compilation in order to run it.
- Interpreted, is a programming language that doesn't require compilation in order to run.
- Multi-paradigm, a way to classify programming languages.

"Common programming paradigms include:
imperative which allows side effects, functional which disallows side effects, declarative which does not state the order in which operations execute, object-oriented which groups code together with the state the code modifies, procedural which groups code into functions, logic which has a particular style of execution model coupled to a particular style of syntax and grammar, and symbolic programming which has a particular style of syntax and grammar."

The control flow is present in many programming languages and JavavScript obeys the same rule.
It allow as the name implies it controls a script's flow of execution. This is widely used in user input scenarios where conditions are set based on input.

Control Statements:


IF

Executes a block of code within it if a condition is true, it only executes once

output// x is greater than 4

Else executes the alternative script when the undesired outcome is obtained.

            console.log("x is not greater than 4");
          }

ELSE IF

Is used to define a sequence of options.

            console.log("x is greater than 5");

output// x is equal to 5

output//
It is a T-shirt day

1|3b - Call Stack

Thread wise, Javascript can only handle one task at a time and being a single threaded and concurrent language a solution had to be put in place to handle more tasks. A system that uses a combination of stacks, heaps and queues was put in place in order to run more than one stack at a time.


The call stack keeps track of all function calls specially in multiple scripts calling. A record of the current function being run and the next one is maintained. A called function is pushed into the stack and a return will be popped off. Stack storing requires space in the computer's memory and if it take more space than allocated a "out of stack" or "too much recursion" error will be thrown out. The order functions are being called in a first in last out basis.

Heaps


The heap is where objects and variable's memory is allocated.

Queue


A queue follows the logic of first in first out, it maintains the order in which tasks are added to the queue and how they should be processed. Enqueue adds elements to the beginning or end of the queue. Dequeue pops out elements from the beginning of the queue. Length is the length of the queue.



1|3c - Closure

Closure defines the ability to reference a specific instance of a local variable outside the function it is enclosed in. The variable instance can still be access even if the parent function has completed executing. These local variables are re-created on each function call and each call is unique, see example 2. Objects in a closure are given data privacy as functions defined outside the closure won't be able to access enclosed objects.

The child function has access to the parent scope (variables and parameters). The parent's function parameter acts a local variable, see example 3. 

          
            //Example 1
            function catalog() {
              var book = 'Javascript';
              function displayBook() {
                alert(book);
              }
              return displayBook;
            }

            var myCat = catalog();
            myCat();
          

          
            //Example 2
            function wrapValue(n){
              var localVariable = n;
              return function(){
                return localVariable;
              };
            }

            var wrap1 = wrapValue(100);
            console.log(wrap1());
            //ouput: 100

            var wrap2 = wrapValue(200);
            console.log(wrap2());
            //ouput: 200
          

          
            //Example 3
            function multiple(factor){
              return function(n){
                return n * factor;
              };
            }

            var twice = multiple(2);
            console.log(twice(5));
            //ouput: 10


1|3d - Recursion

Recursion is a function that calls itself as long the stack is not overflow. A recursion encompasses two cases a base case (returns a value) and recursive case (self-call).

In the example the factorial() function calls itself several times with different arguments to achieve the required result.
The only caveat is that self function calls will slow down the execution of the script compare to a simple loop (speed vs code elegance). 


        
          function func(){
            if(/*base case*/){
              return something;
            }else{/*recursive case*/
              func();
            }
          }

          function factorial(n){
            if(n == 1){
              return n;
            }else{
              return n * factorial(n -1);
            }
          }
          factorial(4);

          //output: 24
